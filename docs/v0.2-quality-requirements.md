# v0.2 Native Quality Requirements

- **대상 릴리스**: v0.2.0-alpha.9
- **작성일**: 2026-06-13

## 1. 핵심 기준

v0.2.0-alpha.9의 품질 기준은 “tmux를 잘 감싸는가”가 아니라 “tuimux daemon이 직접 소유한 PTY terminal, nested split pane, pane resize, paste path가 native terminal처럼 믿고 쓸 수 있는가”다.

## 2. 릴리스 차단 조건

- 기본 `tuimux` 실행이 tmux attach 화면으로 들어가면 실패다.
- tmux가 설치되어 있지 않다는 이유로 기본 실행 또는 doctor가 실패하면 실패다.
- 80x24 host에서 `btop`이 terminal size 오류를 내면 실패다.
- `htop`, `nano` 같은 alternate-screen 앱이 화면 잔상이나 입력 불능 상태에 빠지면 실패다.
- drag selection이 mouse-up 직후 사라지면 실패다.
- selection 상태 Ctrl-C가 clipboard copy 대신 child process interrupt로 처리되면 실패다.
- UI detach 후 같은 session에 재attach했을 때 shell process state가 사라지면 실패다.
- split pane 후 입력, cursor, mouse 좌표가 다른 pane으로 새면 실패다.
- nested split이 기존 split 구조를 덮어쓰면 실패다.
- pane resize 후 PTY size와 UI rect가 서로 다르면 실패다.
- split pane에서 drag selection이 다른 pane으로 focus를 바꾸거나 mouse-up 직후 사라지면 실패다.
- host paste가 paste event로 처리되지 않아 큰 붙여넣기가 개별 key 입력처럼 흔들리면 실패다.
- macOS에서 daemon socket path가 길어서 조용히 in-process fallback으로 내려가면 실패다.
- macOS ARM release build와 installer 검증이 실패하면 릴리스하지 않는다.

## 3. 필수 검증

```sh
cargo fmt -- --check
cargo test --quiet
TERM=xterm-256color COLORTERM=truecolor cargo run --quiet -- --doctor
TERM=dumb cargo run --quiet -- --doctor
cargo build --release --locked --target aarch64-apple-darwin
```

수동 smoke:

- `tuimux --session smoke`
- shell에서 `printf 'tuimux-native-smoke\n'`
- shell에서 `export TUIMUX_PERSIST_MARK=alive`
- UI 종료 후 같은 session 재attach
- shell에서 `echo $TUIMUX_PERSIST_MARK`
- 출력 텍스트 drag selection
- Ctrl-C
- host shell에서 `pbpaste`
- navigation mode에서 `|` 또는 `v`로 right split
- navigation mode에서 `-` 또는 `h`로 down split
- navigation mode에서 arrow key로 active pane resize
- `Tab`으로 pane cycle, `x`로 active pane kill
- host paste가 active pane shell에 전달되는지 확인
- `btop`, `htop`, `nano`, `llmfit --help`

## 4. 허용되는 알파 한계

- session은 daemon memory 안에서만 유지된다. daemon 종료나 reboot 후 복구는 보장하지 않는다.
- tmux layout string import/export와 command language 호환성은 아직 없다.
- tmux command/plugin/config 호환성은 없다.
- Windows/Linux asset은 이번 프리릴리즈 범위 밖이다.

## 5. 다음 품질 목표

- tmux layout string 호환성.
- daemon restart 후 session metadata 복구.
- 동시 multi-attach 정책.
- macOS x86_64, Linux, Windows 확대 검증.
