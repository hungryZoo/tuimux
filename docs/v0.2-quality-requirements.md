# v0.2 Native Quality Requirements

- **대상 릴리스**: v0.2.0-alpha.33
- **작성일**: 2026-06-13

## 1. 핵심 기준

v0.2.0-alpha.33의 품질 기준은 “tmux를 잘 감싸는가”가 아니라 “tuimux daemon이 직접 소유한 PTY terminal, visible terminal-mode chrome, alternate-screen behavior, window-list navigation, scrollback, paste path, OSC title/clipboard, child process lifecycle, concurrent attach가 native terminal처럼 믿고 쓸 수 있는가”다.

## 2. 릴리스 차단 조건

- 기본 `tuimux` 실행이 tmux attach 화면으로 들어가면 실패다.
- 넓은 기본 terminal mode에 boxed right rail, STATUS panel, `scroll:<count>`가 보이지 않으면 실패다.
- terminal mode가 top/bottom status bar로 child PTY 높이를 줄이면 실패다.
- tmux가 설치되어 있지 않다는 이유로 기본 실행 또는 doctor가 실패하면 실패다.
- 80x24 host에서 `btop`이 terminal size 오류를 내면 실패다.
- `btop`이 `CSI row;col f` HVP cursor-position sequence를 사용했을 때 화면이 순차 출력처럼 무너지면 실패다.
- `htop`, `nano` 같은 alternate-screen 앱이 화면 잔상이나 입력 불능 상태에 빠지면 실패다.
- drag selection이 mouse-up 직후 사라지거나 화면 하이라이트가 남지 않으면 실패다.
- selection 상태 Ctrl-C가 clipboard copy 대신 child process interrupt로 처리되면 실패다.
- daemon `SelectedText` request가 화면 좌표의 텍스트를 반환하지 못하거나 selection snapshot이 선택 영역을 강조하지 못하면 실패다.
- macOS PTY UI smoke에서 mouse-up 후 reverse-video selection highlight가 관측되지 않거나 Ctrl-C가 foreground child SIGINT로 관측되거나 host bracketed paste 출력이 child PTY에 나타나지 않거나 child bracketed paste wrapper가 보존되지 않으면 실패다.
- child가 mouse tracking을 켠 상태에서 normal mouse가 child로 전달되지 않거나 Shift-drag가 tuimux selection override 대신 child로 누수되면 실패다.
- parent `NO_COLOR=1` 환경 때문에 child가 명시적으로 출력한 truecolor foreground/background/default reset이 실제 TUI output에서 사라지면 실패다.
- host resize 후 active child process가 새 rows/cols와 `SIGWINCH`를 관측하지 못하면 실패다.
- 긴 terminal row 뒤에 짧은 row를 렌더했을 때 이전 frame glyph가 남거나 alternate-screen text가 primary screen/scrollback에 누수되면 실패다.
- UI detach 후 같은 session에 재attach했을 때 shell process state가 사라지면 실패다.
- navigation mode에서 `Tab`/arrow key가 window-list 전환 대신 deprecated pane 조작으로 동작하면 실패다.
- navigation mode에서 `n`/`x`로 window를 만들고 닫을 수 없으면 실패다.
- child OSC 0/1/2 title이 오른쪽 window list에 반영되지 않으면 실패다.
- child OSC 52 clipboard copy가 macOS system clipboard에 반영되지 않거나 child OSC 52 paste query가 clipboard text를 PTY response로 돌려받지 못하면 실패다.
- child shell이 `exit` 직전 출력한 화면을 한 snapshot도 보여주지 못하거나, 종료 뒤 stale terminal 화면이 window list에 계속 남거나, 마지막 window replacement shell이 새 명령을 받지 못하면 실패다.
- legacy split hotkey가 default UI에서 새 pane을 만들면 실패다.
- native mux core에 split layout tree나 split/resize/kill pane 함수가 남아 있으면 실패다.
- shell output이 화면 높이를 넘은 뒤 실제 TUI에서 mouse wheel, `PageUp`, `Home`, `End`로 과거 화면을 보고 bottom으로 돌아올 수 없으면 실패다.
- scrollback 중 paste를 보냈을 때 live bottom으로 돌아와 active shell에서 실행되지 않으면 실패다.
- host paste가 paste event로 처리되지 않아 큰 붙여넣기가 개별 key 입력처럼 흔들리면 실패다.
- 열린 client 때문에 다른 attach, command, shutdown request가 막히면 실패다.
- macOS에서 daemon socket path가 길어서 조용히 in-process fallback으로 내려가면 실패다.
- macOS ARM release build와 installer 검증이 실패하면 릴리스하지 않는다.

## 4. 허용되는 알파 한계

- session은 daemon memory 안에서만 유지된다. daemon 종료나 reboot 후 복구는 보장하지 않는다.
- tmux command/plugin/config 호환성은 없다.
- split-pane UX는 deprecated이며 native mux core에는 split layout state가 없다. 기본 제품 방향은 window-list workflow다.
- Windows/Linux asset은 이번 프리릴리즈 범위 밖이다.
